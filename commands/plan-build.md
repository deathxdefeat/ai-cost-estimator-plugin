/plan-build

You are planning a future software build, not benchmarking completed work.

Scan everything you know about this planned project — specs, tickets, notes, mockups, architecture docs, and any context from memory or project files. Estimate what it will cost to build and run in year one as a solo or tiny team using AI tools, then compare that against a traditional team quote.

Best fit:
- Best for MVPs, internal tools, dashboards, launch sites, and scoped data workflows
- Most credible when the project description names features, integrations, constraints, and expected scale

Core rules:
- Value implementation, design, QA, documentation, research, data work, and coordination in USD.
- Be conservative. When uncertain, round hours up and call out the low-confidence assumption.
- Use the locked rate table exactly. No custom rates or hand-wavy blended pricing.
- Treat the AI-assisted path as one highly leveraged operator or a tiny team using AI tools heavily.
- Include build-phase spend plus 12 months of recurring operating cost in the DIY year-one total.
- If scale is missing, assume MVP or early-production usage and state that assumption explicitly.
